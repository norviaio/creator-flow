# creator-flow

**creator-flow** は、個人制作者や小規模チーム向けの **コンテンツ制作進行管理ツール**です。  
制作物（作品）と、制作に必要なタスクを管理し、進捗状況をシンプルに可視化することを目的としています。

> 本プロジェクトは転職活動用ポートフォリオとして作成しています。

---

## 目的
- 制作中のタスクが散らばる問題を減らす
- 進捗（いま何が止まっているか）を一目で分かるようにする
- Auth + DB + UI + Deploy を含むフルスタック構成を、1ヶ月で完成させる

## 対象ユーザー
- 個人制作者（動画・漫画・記事・配信企画など）
- 小規模チーム（2〜5人程度を想定、ただしMVPは1人運用で成立）

## 用語
- **作品（Project）**：制作対象の単位（例：動画1本、漫画1話、記事1本）
- **タスク（Task）**：作品に紐づく作業（例：台本、編集、サムネ作成）

---

## MVP機能（1ヶ月で作る範囲）
### 認証
- ログイン / ログアウト（Supabase Auth想定）
- ログイン済みユーザーのみ操作できる
- 通常ユーザー：自分が作成した作品/タスクのみ閲覧・編集可能
- 管理者（admin）：全ユーザーの作品/タスクを閲覧可能（Supabaseの `admins` テーブルで user_id を管理）

### 作品（Project）
- 一覧表示
- 作成 / 編集 / 削除
- 作品ステータス管理：`active` / `completed`

### タスク（Task）
- 作品に紐づくタスクの一覧表示
- 作成 / 編集 / 削除
- タスクステータス変更（テーブル上でSelect変更）
- ステータスで絞り込み（フィルタ）

#### タスクステータス（固定）
- `backlog`
- `in_progress`
- `review`
- `done`

---

## やらないこと（スコープ外）
- 決済
- 外部API連携（YouTube/Twitch等）
- カレンダー表示、ドラッグ&ドロップ
- 通知（メール/Push）
- 実在のVTuber名・作品名・ロゴ・画像の利用（著作権回避のため）

---

## 技術スタック（予定）
- **Design**：Figma
- **Frontend**：Next.js / React / TypeScript / Tailwind CSS
- **DB & Auth**：PostgreSQL（Supabase想定）
- **Infra**：Vercel
- **Version Control**：GitHub

## 画面一覧（MVP）
1. **ログイン**
   - メールアドレスでログイン
   - 未ログイン時は他画面にアクセス不可

2. **ダッシュボード**
   - 自分の全タスクを横断的に一覧表示
   - ステータスで絞り込み

3. **作品一覧**
   - 作品（Project）の一覧表示
   - 作成 / 編集 / 削除

4. **作品詳細**
   - 作品情報（タイトル / 説明 / ステータス）
   - 紐づくタスク一覧

5. **タスク作成 / 編集**
   - タスク名
   - ステータス選択

## セットアップ（予定）
※ 実装完了後に更新予定

```bash
# リポジトリをクローン
git clone git@github.com:norviaio/creator-flow.git
cd creator-flow

# 依存関係のインストール
npm install

# 環境変数の設定（例）
# .env.local を作成し、Supabaseのキーを設定
# NEXT_PUBLIC_SUPABASE_URL=xxxxx
# NEXT_PUBLIC_SUPABASE_ANON_KEY=xxxxx

# 開発サーバー起動
npm run dev
```
