# creator-flow

**creator-flow** は、個人制作者や小規模チーム向けの **コンテンツ制作進行管理ツール**です。  
制作物（作品）と、制作に必要なタスクを管理し、進捗状況をシンプルに可視化することを目的としています。

> 本プロジェクトは転職活動用ポートフォリオとして作成しています。

---

## 目的

- 制作中のタスクが散らばる問題を減らす
- 進捗（いま何が止まっているか）を一目で分かるようにする
- Auth + DB + UI + Deploy を含むフルスタック構成を、1 ヶ月で完成させる

## 対象ユーザー

- 個人制作者（動画・漫画・記事・配信企画など）
- 小規模チーム（2〜5 人程度を想定、ただし MVP は 1 人運用で成立）

## 用語

- **作品（Project）**：制作対象の単位（例：動画 1 本、漫画 1 話、記事 1 本）
- **タスク（Task）**：作品に紐づく作業（例：台本、編集、サムネ作成）

---

## MVP 機能（1 ヶ月で作る範囲）

### 認証

- ログイン / ログアウト（Supabase Auth）
- ログイン済みユーザーのみ操作できる
- 通常ユーザー：自分が作成した作品/タスクのみ閲覧・編集可能
- 管理者（admin）：全ユーザーの作品/タスクを閲覧可能  
  （Supabase の `admins` テーブルで user_id を管理）
  ※ 設計のみ実施。実装は今後の拡張予定

### 作品（Project）

- 一覧表示
- 作成
- 詳細表示
- 名称編集
- ステータス管理（active / completed）

### タスク（Task）

- 一覧表示
- 作成
- タスク名編集
- ステータス更新
- 削除
- ステータスで絞り込み（フィルタ）

#### タスクステータス（固定）

- `backlog`
- `in_progress`
- `review`
- `done`

## やらないこと（スコープ外）

- 決済
- 外部 API 連携（YouTube/Twitch 等）
- カレンダー表示、ドラッグ&ドロップ
- 通知（メール/Push）
- 実在の VTuber 名・作品名・ロゴ・画像の利用（著作権回避のため）

## 設計・実装のポイント

### フルスタック構成

- Next.js（App Router）をフロントエンドに採用
- Supabase を BaaS として利用し、認証・DB・RLS を一体で管理
- API を最小限にし、UI とデータの責務を明確に分離

### Row Level Security（RLS）

- projects / tasks は、ログインユーザー自身のデータのみ操作可能
- 認可をフロントではなく DB レベルで制御することで安全性を担保

### 型設計

- DB の型（Row）とフロント用の型を分離
  - TaskRow：DB 構造（snake_case）
  - Task：UI 用（camelCase）
- any を使わず、型安全に実装

### 段階的な開発

- 初期は UI と mock データで設計を固める
- その後 Supabase 実データに段階的に移行

## 画面一覧（MVP）

1. **ログイン**

   - メールアドレスでログイン
   - 未ログイン時は他画面にアクセス不可

2. **ダッシュボード（/）**

   - ログイン後のトップページ
   - プロジェクト管理画面への導線を提供
   - 現在はアプリ概要とナビゲーションを中心に実装
   - タスク横断表示は今後の拡張予定

3. **作品一覧**

   - 作品（Project）の一覧表示
   - 作成 / 編集 / 削除

4. **作品詳細**

   - 作品情報（タイトル / 説明 / ステータス）
   - 紐づくタスク一覧

5. **タスク作成 / 編集（作品詳細内）**
   - タスク名（クリックで編集）
   - ステータス選択（Select で変更）
   - 削除

## セットアップ（暫定）

※ 実装完了後に更新予定

```bash
# リポジトリをクローン
git clone git@github.com:norviaio/creator-flow.git
cd creator-flow

# 依存関係のインストール
npm install

# 環境変数の設定（例）
# .env.local を作成し、Supabaseのキーを設定
# NEXT_PUBLIC_SUPABASE_URL=xxxxx
# NEXT_PUBLIC_SUPABASE_ANON_KEY=xxxxx

# 開発サーバー起動
npm run dev
```
